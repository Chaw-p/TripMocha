1단계: 로컬 변경 사항 임시 저장 (Stash)

현재 작업 중이던 커밋되지 않은 모든 변경 사항을 스택에 임시로 저장합니다.


git stash


설명: 이 명령을 실행하면 로컬 작업 디렉터리가 깨끗해지며, Git은 이제 원격 내용을 가져올 준비가 됩니다.

2단계: 원격 저장소의 최신 내용 가져오기 (Pull)

이제 원격 저장소(origin/main 또는 origin/master)의 최신 내용을 로컬 브랜치로 가져와 안전하게 병합합니다.


git pull origin main


주의: 만약 사용 중인 브랜치 이름이 main이 아니라면, 해당 이름으로 바꿔주세요.

3단계: 임시 저장했던 로컬 내용 다시 적용 (Stash Pop)

1단계에서 임시로 저장해 둔 변경 사항을 다시 현재 작업 브랜치에 적용합니다.


git stash pop


결과 A (충돌 없음): 로컬 변경 사항이 성공적으로 적용됩니다.

결과 B (충돌 발생): 로컬 변경 사항과 원격 변경 사항 사이에 충돌이 발생하면 터미널에 충돌 파일 목록이 표시됩니다.

4단계: (선택 사항) 충돌 해결 및 최종 커밋

3단계에서 충돌이 발생했다면, 다음 순서에 따라 수동으로 해결해야 합니다.

충돌 파일 확인: IDE(예: VS Code)를 열어 충돌이 발생한 파일을 확인하고 <<<<<<<, =======, >>>>>>> 표시를 기준으로 원하는 코드를 선택하거나 수정하여 충돌을 해결합니다.

해결된 파일 스테이징: 충돌을 해결한 파일을 Git 스테이징 영역에 추가합니다.


git add . 


최종 커밋: 모든 변경 사항을 커밋합니다.


git commit -m "Merge conflicts resolved and local work applied"


5단계: 원격 저장소에 최종 푸시 (Push)

모든 병합 및 충돌 해결이 완료되었다면, 최종 결과를 원격 저장소에 업로드합니다.


git push origin main


이 단계를 모두 거치면 Git 오류가 해결되고 프로젝트를 계속 진행할 수 있습니다.
